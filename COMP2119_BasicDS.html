1) What COMP2119 expects you to say

ADT vs Implementation (marks are strict)

  • ADT (abstract data type) = operations + behavior. Example: Stack supports push, pop, top with LIFO semantics.
  • Implementation = how it’s stored (array, linked list, two stacks, etc.).
  • Full‑mark habit: when you claim an operation is \(O(1)\), say whether it is worst‑case or amortized.
Exam wording translator:
  • “Give an algorithm” → show steps/pseudocode + base cases + complexity.
  • “Trace the operations” → show structure state after each step OR output sequence.
  • “Explain why \(O(1)\)” → give a short amortized/charging argument.

2) Arrays & Dynamic Arrays (vector-style)

Static array

  • Random access \(A[i]\) is \(O(1)\).
  • Insert/delete in the middle requires shifting: \(O(n)\).
  • Best for: frequent random access, fixed size.

Dynamic array (like C++ vector)

  • Stores capacity and size.
  • When size==capacity, allocate bigger array (often double) and copy.
  • Append is amortized \(O(1)\), but a single append can be \(O(n)\) during resize.
Common exam question: “Why is append amortized \(O(1)\)?” → you must give a charging/potential proof (see Section 6). 
// Typical dynamic array fields
size   // number of elements stored
cap    // allocated length
arr    // pointer to memory of length cap

append(x):
  if size == cap:
    cap = max(1, 2*cap)
    newArr = allocate(cap)
    copy arr[0..size-1] to newArr
    arr = newArr
  arr[size] = x
  size++

Worst-case vs amortized: say it correctly

  • Worst-case append: \(O(n)\) (when resizing/copying).
  • Amortized append over \(m\) appends: \(O(1)\).
  • Space overhead: capacity can be up to ~2×size (constant factor).

3) Linked Lists (Singly & Doubly) — pointer discipline

Node models you must recognize

// singly
struct Node {
  int data;
  Node* next;
};

// doubly
struct DNode {
  int data;
  DNode* prev;
  DNode* next;
};

Core invariants (write them in exam answers)

  • Head points to the first node, or NULL if empty.
  • Each node’s next points to the next node, last node’s next is NULL.
  • Doubly list: for any interior node x: x.prev.next == x and x.next.prev == x.

Singly list: insert at head (O(1))

insertHead(head, x):
  n = new Node(x)
  n.next = head
  head = n
  return head

Singly list: delete first occurrence of value (O(n))

deleteValue(head, x):
  if head == NULL: return head
  if head.data == x:
    head = head.next
    return head
  prev = head
  cur  = head.next
  while cur != NULL and cur.data != x:
    prev = cur
    cur = cur.next
  if cur != NULL:
    prev.next = cur.next
  return head
Big trap: In a singly list, to delete cur you need prev. If you don’t store it, you cannot fix the link.

In-place reverse of singly list (high-frequency)

Memorize the “prev/cur/nxt” pattern. If you can write this from memory, you’re safe.

reverse(head):
  prev = NULL
  cur  = head
  while cur != NULL:
    nxt = cur.next
    cur.next = prev
    prev = cur
    cur  = nxt
  return prev
Correct complexity statement: time \(\Theta(n)\), extra space \(\Theta(1)\).

Cycle detection (Floyd) — classic

hasCycle(head):
  slow = head
  fast = head
  while fast != NULL and fast.next != NULL:
    slow = slow.next
    fast = fast.next.next
    if slow == fast:
      return true
  return false

Doubly list: delete given node pointer in \(O(1)\)

deleteNode(head, tail, x):
  if x == NULL: return
  if x.prev != NULL: x.prev.next = x.next
  else:              head = x.next       // x was head
  if x.next != NULL: x.next.prev = x.prev
  else:              tail = x.prev       // x was tail
Trap: if x is head or tail, you must update the external head/tail pointer. Students lose marks here.

4) Stacks — tracing and the classic patterns

Stack operations

  • push(x), pop(), top(), isEmpty() are \(O(1)\).
  • Array implementation: use an integer top index.
  • Linked list implementation: push/pop at head.

How to trace stack questions quickly

  • Draw the stack with top on the right.
  • Only pop() produces output.
  • If the question specifies behavior on empty pop, state “underflow”.

Bracket matching (must be automatic)

  1. Push opening brackets ( [ {.
  2. On a closing bracket: stack must not be empty and top must match type.
  3. At the end: stack must be empty.

Infix → Postfix (exam-friendly explanation)

  • Operands go directly to output.
  • Operators: pop while stack top has higher or equal precedence (for left-associative ops), then push current operator.
  • Parentheses: push '('; when ')' appears, pop until '(' is removed.
Associativity trap: For exponentiation (right-associative), you do NOT pop on equal precedence.

5) Queues & Circular Queues (array implementation is exam-favorite)

Queue basics

  • FIFO: first in, first out.
  • enqueue at rear, dequeue at front.
  • Target complexity: \(O(1)\) per operation.

Circular array queue (the exam trap zone)

You MUST state the convention you use.

Recommended convention:
  • front points to the first element.
  • rear points to the next free slot.
  • Advance by \((i+1)mod m\).

Empty vs Full (two valid solutions)

  • Option A (store size): empty if size=0, full if size=m.
  • Option B (one-empty-slot): empty if front==rear, full if (rear+1)%m == front.

Pseudocode (Option B: one-empty-slot)

// capacity m, array Q[0..m-1]
isEmpty(): front == rear
isFull():  (rear + 1) % m == front

enqueue(x):
  if isFull(): error
  Q[rear] = x
  rear = (rear + 1) % m

dequeue():
  if isEmpty(): error
  x = Q[front]
  front = (front + 1) % m
  return x
Hard trap: Option B stores at most \(m-1\) items. You must say this.

Queue using two stacks (high yield)

enqueue(x):
  push(in, x)

dequeue():
  if empty(out):
    while not empty(in):
      push(out, pop(in))
  return pop(out)
Correct statement: worst-case dequeue can be \(O(n)\), but amortized cost per operation is \(O(1)\).

6) Amortized proofs — the exact short answers

A) Dynamic array doubling ⇒ amortized \(O(1)\) append

  • When capacity doubles from \(k\) to \(2k\), you copy exactly \(k\) items.
  • Between these two resizes, you perform at least \(k\) appends.
  • Charge each append a constant “credit” to pay for future copying.
  • Total cost over \(m\) appends is \(O(m)\) ⇒ amortized \(O(1)\).
One-line bound: total copied items \(\le 1 + 2 + 4 + \dots + 2^{\lfloor \log m floor} < 2m\).

B) Two-stack queue ⇒ amortized \(O(1)\)

  • Each element: pushed to in once.
  • Moved from in to out at most once (one pop + one push).
  • Popped from out once when dequeued.
  • So ≤ 4 stack ops per element ⇒ total \(O(m)\) for \(m\) queue operations ⇒ amortized \(O(1)\).
Important wording: amortized \(O(1)\) ≠ every operation \(O(1)\). It’s the average over a sequence.

7) Hard traps (this is where marks disappear)

Linked list traps

  • Losing head by overwriting pointers without saving them.
  • Dereferencing NULL: accessing cur.next when cur might be NULL.
  • Deleting in singly list without storing predecessor.
  • Doubly list deletion: forgetting to update both directions, or head/tail pointers.

Circular queue traps

  • Not stating the empty/full rule (size vs one-empty-slot).
  • Mixing conventions: rear as “last element” vs “next free slot”. Pick one.
  • Forgetting modulo update.

Tracing traps

  • Output only on pop/dequeue, not on push/enqueue.
  • If asked: explicitly state “underflow/overflow” behavior.
Strict exam habit: write invariants before code: “I assume rear points to next free slot”.

8) Hard practice set (COMP2119-style) — with answer key

A) Trace (stack)

Stack initially empty. Operations:

  1. push(12), push(3), push(5), pop(), push(26), push(2), pop(), pop(), push(85), pop(), pop()
  2. Write (i) pop output sequence (ii) final stack content (top on right).

B) Trace (queue)

Queue initially empty. Operations:

  1. enqueue(12), enqueue(3), enqueue(5), dequeue(), enqueue(26), enqueue(2), dequeue(), dequeue(), enqueue(85), dequeue(), dequeue()
  2. Write (i) dequeue output sequence (ii) final queue content (front→rear).

C) Circular queue reasoning

  1. Explain why front==rear is ambiguous if you only store indices.
  2. Give 2 fixes (size variable and one-empty-slot) with full/empty conditions.
  3. Capacity \(m\) using one-empty-slot: what is max stored elements?

D) Linked list coding

  1. Write in-place reverse of a singly linked list.
  2. Write Floyd cycle detection.
  3. Delete a node in a doubly linked list given a pointer to it (handle ends).

E) Amortized proofs

  1. Prove dynamic array append is amortized \(O(1)\) under doubling (short).
  2. Prove two-stack queue is amortized \(O(1)\) (short).

Answer key (open only after attempting)

Show answers

A)

  • Pop outputs: 5, 2, 26, 85, 3
  • Final stack: [12]

B)

  • Dequeue outputs: 12, 3, 5, 26, 2
  • Final queue: empty

C)

  • Ambiguity: front==rear could mean empty or full.
  • Fix 1 (size): empty size=0, full size=m.
  • Fix 2 (one-empty-slot): empty front==rear, full (rear+1)%m==front.
  • Max stored: \(m-1\).

D)

  • Reverse: prev/cur/nxt loop; time \(\Theta(n)\), space \(\Theta(1)\).
  • Cycle: Floyd; time \(\Theta(n)\), space \(\Theta(1)\).
  • Doubly delete: update prev.next and next.prev if they exist; if node is head/tail update external pointers.

E)

  • Dynamic array: total copies form geometric series <(\(2m\)); total \(O(m)\); amortized \(O(1)\).
  • Two stacks: each element causes ≤4 stack ops total; total \(O(m)\); amortized \(O(1)\).
