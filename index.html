1) Asymptotic notation — what to write for full marks

Big‑O (upper bound)

\(f(n)=O(g(n))\) means:

\(\exists c>0,\exists n_0\) such that \(\forall n\ge n_0:\ 0\le f(n)\le c\,g(n)\).

Big‑Omega (lower bound)

\(f(n)=\Omega(g(n))\) means:

\(\exists c>0,\exists n_0\) such that \(\forall n\ge n_0:\ 0\le c\,g(n)\le f(n)\).

Big‑Theta (tight bound)

\(f(n)=\Theta(g(n))\) means both upper and lower bounds:

\(\exists c_1,c_2>0,\exists n_0\) such that \(\forall n\ge n_0:\ c_1 g(n)\le f(n)\le c_2 g(n)\).

little‑o and little‑omega (strictly smaller / bigger)

little‑o \(f(n)=o(g(n))\) iff \(\displaystyle \lim_{n\to\infty}\frac{f(n)}{g(n)}=0\).
little‑omega \(f(n)=\omega(g(n))\) iff \(\displaystyle \lim_{n\to\infty}\frac{f(n)}{g(n)}=\infty\).
Exam-saver: If you see \(o(\cdot)\) or \(\omega(\cdot)\), immediately form the ratio and take the limit.

Log base change (always worth marks)

\(\log_a n=\frac{\log_b n}{\log_b a}\). So any constant base change is a constant factor:

\(\log_2 n = \Theta(\log_{10} n)\) and generally \(\log_a n = \Theta(\log n)\) for any constant \(a>1\).

Safe transforms & unsafe transforms

Given Safe conclusion Common trap
\(f=\Theta(g)\), \(f,g\ge 2\) eventually \(\log f = \Theta(\log g)\) Because \(\log(cg)=\log g + O(1)\)
\(f=O(g)\), \(f,g\ge 2\) eventually \(\log f = O(\log g)\) \(\log f \le \log c + \log g \le (\log c + 1)\log g\) for large \(n\)
\(f=O(g)\) NOT necessarily \(2^f = O(2^g)\) Trap: \(f=2n, g=n\) ⇒ \(2^{2n}\notin O(2^n)\)

Top 8 “don’t lose marks” rules

  1. Always state \(c>0\), \(n_0\) in proofs.
  2. When combining O-inequalities, use \(n\ge \max(n_0,n_1)\).
  3. Log base changes are constant factors.
  4. \((n+1)! \neq \Theta(n!)\). Ratio is \(n+1\).
  5. Any \(n^\epsilon\) beats \((\log n)^k\) eventually.
  6. Any \(a^n\) beats \(n^k\) eventually (for \(a>1\)).
  7. Master Theorem: compare \(f(n)\) to \(n^{\log_b a}\).
  8. For nested loops, convert to sums.

2) Growth ordering — what dominates what

Standard growth ladder (memorize)

\(\displaystyle 1 \ll \log\log n \ll \log n \ll n^\epsilon \ll n \ll n\log n \ll n^2 \ll n^k \ll a^n \ll n! \ll n^n\) (for any constants \(\epsilon>0, k>1, a>1\))

Fast ranking rules

TypeRuleExample
Polynomials Highest power wins \(n^5+4n^3+2n+\log^2 n = \Theta(n^5)\)
Logs \(\log^k n \ll n^\epsilon\) for any \(\epsilon>0\) \(\log^2 n = o(n^{0.0001})\)
Exponentials Bigger base wins \(2^n = o(3^n)\)
Factorial vs exponential \(a^n = o(n!)\) for any constant \(a>1\) \(2^n = o(n!)\)
Constant multiples Ignore constants inside Θ \(2\cdot 3^{n+2} = 18\cdot 3^n = \Theta(3^n)\)
Hard-but-common comparisons (quick answers)
  • \(\log_5 n\) vs \(\log_2 n\): Θ-equal.
  • \((n+1)!\) vs \(n!\): not Θ-equal. \((n+1)! = \Theta(n\cdot n!)\).
  • \(\log(n!) = \Theta(n\log n)\).
  • \(2^{\log_2 n} = n\). More generally \(a^{\log_b n}=n^{\log_b a}\).
  • \(n^{1.01}\) vs \(n\log n\): \(n^{1.01}\) dominates.

3) Proof templates — write like an A-range student

Template A: Prove \(f(n)=O(g(n))\)

Goal: find c>0 and n0 such that for all n>=n0: 0 <= f(n) <= c*g(n).
Steps:
1) Start with f(n) <= ...
2) Bound smaller terms by the biggest term (for n large).
3) Choose constants (c, n0) explicitly.
4) State: Therefore f(n)=O(g(n)).
Example: Prove \(5n+4=O(n)\). For \(n\ge 4\): \(5n+4 \le 5n+n=6n\). Choose \(c=6, n_0=4\).

Template B: Disprove \(f(n)=O(g(n))\)

Goal: show NO constants can make it true.
Write:
Assume for contradiction there exist c>0, n0 such that for all n>=n0: f(n) <= c*g(n).
Then choose a specific n (often n=2^k, n=k^2, etc.) that breaks it.
Show f(n)/(g(n)) -> infinity, so inequality fails for large n.
Classic disproof: \(n^2 \notin O(n\log n)\). Let \(n=2^k\). Then need \(2^{2k} \le c\cdot 2^k \cdot k\) ⇒ \(2^k/k \le c\), but \(2^k/k\to\infty\).

Template C: Prove \(\Theta\)

Do both sides:

Upper bound: f(n) <= c2*g(n) for n>=n0
Lower bound: c1*g(n) <= f(n) for n>=n0
Conclude: f(n) = Theta(g(n))
Most common Θ-proof trick: factor out the dominant term and bound the remaining factor between two constants. Example: \(\sqrt{8n^2+2n-16} = n\sqrt{8+\frac{2}{n}-\frac{16}{n^2}} = \Theta(n)\).

Transitivity (always show max)

If \(f=O(g)\) with constants \((c_0,n_0)\) and \(g=O(h)\) with \((c_1,n_1)\), then for \(n\ge \max(n_0,n_1)\): \(f \le c_0 g \le c_0c_1 h\). Choose \(c=c_0c_1\), \(n_2=\max(n_0,n_1)\).

4) Loop analysis — convert loops into sums

Golden rule

Nested loops = nested sums. Your job is to simplify the sum to Θ.

Common patterns

Code patternHow to thinkΘ
for i=1..n: for j=1..n: O(1) \(\sum_{i=1}^n \sum_{j=1}^n 1 = n^2\) \(\Theta(n^2)\)
for i=1..n: for j=1..i: O(1) \(\sum_{i=1}^n i\) \(\Theta(n^2)\)
for i=1..n: for j=n..i: O(1) \(\sum_{i=1}^n (n-i+1)\) \(\Theta(n^2)\)
for i=1..n: j=1; while j<=n: j*=2 doubling loop runs \(\log n\) times \(\Theta(n\log n)\)

Hard pattern: shrinking while loop totals to linear

If inner loop starts at \(j=i\) and doubles until \(n\), total work can be linear: \(\sum_{i=1}^n \Theta(\log(n/i)+1)=\Theta(n)\). (Bucket argument: half the i’s do 1 step, quarter do 2 steps, etc.)
Cheat sums you should know
  • \(\sum_{i=1}^n 1 = n\)
  • \(\sum_{i=1}^n i = n(n+1)/2 = \Theta(n^2)\)
  • \(\sum_{i=1}^n i^2 = \Theta(n^3)\)
  • \(\sum_{i=0}^{\log n} 1 = \Theta(\log n)\)
  • \(\sum_{i=0}^{\log n} (\log n - i)=\Theta(\log^2 n)\)
  • \(\sum_{t\ge 1} t/2^t = 2\) (constant)

5) Recurrences — the full toolbox

Step 0: Identify the type

FormToolExample

Master Theorem (must memorize)

For \(T(n)=aT(n/b)+f(n)\) with \(a\ge 1, b>1\), compare \(f(n)\) to \(n^{\log_b a}\).
Master Theorem — the 3 cases (exam form)
  • Case 1: If \(f(n)=O(n^{\log_b a-\epsilon})\) for some \(\epsilon>0\), then \(T(n)=\Theta(n^{\log_b a})\).
  • Case 2: If \(f(n)=\Theta(n^{\log_b a}\log^k n)\) for \(k\ge 0\), then \(T(n)=\Theta(n^{\log_b a}\log^{k+1} n)\).
  • Case 3: If \(f(n)=\Omega(n^{\log_b a+\epsilon})\) and the regularity condition holds, then \(T(n)=\Theta(f(n))\).
Case 2 is the common killer: If \(T(n)=2T(n/2)+n\log n\), then \(n^{\log_2 2}=n\), and \(f(n)=n\log n\) ⇒ \(T(n)=\Theta(n\log^2 n)\).

Telescoping recurrences

For \(T(n)=T(n-k)+f(n)\): unfold until base case ⇒ sum of \(f(\cdot)\) terms.

Example: \(T(n)=T(n-2)+\log n\). There are \(\approx n/2\) terms, each about \(\log n\) ⇒ \(T(n)=\Theta(n\log n)\).

Log-star recurrence

If \(T(n)=T(\log n)+1\), each step logs the input. Number of steps until constant is \(\log^* n\). So \(T(n)=\Theta(\log^* n)\).

Multiple-branch “level sum” trick

When the subproblem sizes add up to \(n\) (fractions sum to 1), each level often costs \(\Theta(n)\).

If \(T(n)=T(an)+T((1-a)n)+n\) with \(0<a<1\), then work per level \(\approx n\) and height \(\Theta(\log n)\) ⇒ \(T(n)=\Theta(n\log n)\).

6) Hard recurrence patterns (A-range)

Pattern H1: One subproblem + log cost

\(T(n)=T(n/2)+\log n\) ⇒ depth \(\log n\), level costs \(\log n - i\) ⇒ \(\Theta(\log^2 n)\).

Pattern H2: Split fractions that sum to 1

\(T(n)=T(2n/5)+T(3n/5)+n\) ⇒ per-level \(\Theta(n)\), height \(\Theta(\log n)\) ⇒ \(\Theta(n\log n)\).

Pattern H3: Three splits summing to 1

\(T(n)=T(n/2)+T(n/3)+T(n/6)+n\). Fractions sum to 1 ⇒ \(\Theta(n\log n)\).

Pattern H4: Input shrinks by log

\(T(n)=T(\log n)+1\) ⇒ \(\Theta(\log^* n)\).

Pattern H5: Weird shrink like \(T(\sqrt{n})\)

Trick: let \(n=2^{2^k}\). Then \(\sqrt{n}=2^{2^{k-1}}\). This often turns it into a linear recurrence in \(k\). Example: \(T(n)=T(\sqrt{n})+1\) ⇒ \(T(2^{2^k})=T(2^{2^{k-1}})+1\) ⇒ \(\Theta(k)=\Theta(\log\log n)\).

Most common trap list

  • Writing \(\log^n\) when you mean \(\log^2 n\). Always write \(\log^2 n\) or \((\log n)^2\).
  • Mixing \(\Omega\) and \(\omega\). \(\Omega\) is “≥ constant multiple”, \(\omega\) is “ratio → ∞”.
  • Assuming \(f=O(g)\) implies \(2^f = O(2^g)\). Not true.
  • For factorial: \((n+1)! + 2^n = \Theta((n+1)!)\), not \(\Theta(n!)\).
