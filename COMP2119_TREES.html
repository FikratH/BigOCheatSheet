1) Terminology — say it precisely (marks are picky)

Basic definitions

  • Tree: connected, acyclic graph. If it has \(n\) vertices then it has exactly \(n-1\) edges.
  • Rooted tree: one node is the root; every other node has exactly one parent.
  • Child / parent: directed away from the root.
  • Leaf: node with 0 children.
  • Depth(v): number of edges on the path root→v.
  • Height(v): length (edges) of the longest downward path from v to a leaf.
Height convention trap: some questions use height(empty)=−1 and height(leaf)=0, others use height(empty)=0 and height(leaf)=1. Always state the convention you are using before giving a number.

Binary tree types (don’t mix them up)

  • Binary tree: each node has at most 2 children (left/right). Nothing about key ordering.
  • Full binary tree: every internal node has exactly 2 children.
  • Perfect binary tree: full AND all leaves at the same depth.
  • Complete binary tree: all levels full except possibly last, which is filled left-to-right (heap shape).

2) Key properties you should be able to use in proofs

Always-true facts

  • Any tree with \(n\) nodes has \(n-1\) edges.
  • In a rooted tree: \(\sum_{v} ext{children}(v) = n-1\).
  • In a binary tree: number of NULL child pointers is \(n+1\) (useful in pointer-count proofs).

Full binary tree leaves formula (high frequency)

If a non-empty full binary tree has \(I\) internal nodes and \(L\) leaves, then

\[ L = I + 1. \]

Fast proof: Every internal node contributes exactly 2 child-edges so total edges \(E=2I\). But any tree has \(E=n-1=(I+L)-1\) edges. Hence \(2I=I+L-1\Rightarrow L=I+1\).

Bounds with height

  • Max nodes in height \(h\) binary tree: \(2^{h+1}-1\) (if height(leaf)=0).
  • Min nodes in height \(h\) binary tree: \(h+1\) (a chain).
  • Perfect tree: nodes at depth \(d\) is \(2^d\), total is \(2^{h+1}-1\).
Exam line: If a binary tree has \(n\) nodes, then its height is between \(\lfloor \log_2 n floor\) and \(n-1\) (depending on balance).

3) Traversals — do these instantly (and don’t mess up the order)

Binary tree traversals

  • Preorder: root, left, right
  • Inorder: left, root, right
  • Postorder: left, right, root
  • Level-order: BFS by levels (queue)
BST reminder: Only for a BST, inorder outputs sorted keys. For a plain binary tree, inorder is not “sorted”.

DFS templates (recursive)

// Preorder
pre(u):
  if u == NULL: return
  visit(u)
  pre(u.left)
  pre(u.right)

// Inorder
in(u):
  if u == NULL: return
  in(u.left)
  visit(u)
  in(u.right)

// Postorder
post(u):
  if u == NULL: return
  post(u.left)
  post(u.right)
  visit(u)

Level-order (BFS) template

levelOrder(root):
  if root == NULL: return
  Q = empty queue
  enqueue(Q, root)
  while not empty(Q):
    u = dequeue(Q)
    visit(u)
    if u.left  != NULL: enqueue(Q, u.left)
    if u.right != NULL: enqueue(Q, u.right)
Order rule: if the question says “visit children in increasing order / left-to-right”, do exactly that. You lose marks for wrong tie-breaking.

Iterative inorder (very common coding question)

iterInorder(root):
  S = empty stack
  u = root
  while u != NULL or not empty(S):
    while u != NULL:
      push(S, u)
      u = u.left
    u = pop(S)
    visit(u)
    u = u.right

Time \(O(n)\). Stack size \(O(h)\) (height).

4) Reconstruction from traversal orders (A-range topic)

Uniqueness rules (memorize)

  • Preorder + Inorder uniquely determines the binary tree (distinct labels).
  • Postorder + Inorder uniquely determines the binary tree (distinct labels).
  • Preorder + Postorder is not unique in general. It becomes unique if you are also told the tree is full.

Pre + In reconstruction algorithm (the exact exam template)

  1. Root is the next element in preorder.
  2. Find root position in inorder → splits left-subtree inorder and right-subtree inorder.
  3. Recurse on the left range, then on the right range.
Write this clearly: use a hashmap \(pos[x]\) for inorder indices to get total \(O(n)\) time instead of \(O(n^2)\). 
// preorder[preIdx] is next root
build(inL, inR):
  if inL > inR: return NULL
  rootVal = preorder[preIdx]; preIdx++
  k = pos[rootVal]               // inorder index
  root = new Node(rootVal)
  root.left  = build(inL, k-1)
  root.right = build(k+1, inR)
  return root

Common reconstruction marking checklist

  • You must show the inorder split at each step.
  • Preorder consumes roots in order — do not “re-scan” preorder for subtree.
  • State the complexity and the data structure (hashmap) used to reach it.
Fast sanity check: the inorder of your drawn tree must match the given inorder exactly.

5) Standard tree algorithms (write them from memory)

Size, height, leaf count

size(u):
  if u == NULL: return 0
  return 1 + size(u.left) + size(u.right)

height(u):              // height(empty)=-1, height(leaf)=0
  if u == NULL: return -1
  return 1 + max(height(u.left), height(u.right))

countLeaves(u):
  if u == NULL: return 0
  if u.left == NULL and u.right == NULL: return 1
  return countLeaves(u.left) + countLeaves(u.right)

Mirror (swap left/right everywhere)

mirror(u):
  if u == NULL: return
  swap(u.left, u.right)
  mirror(u.left)
  mirror(u.right)

Time \(O(n)\). Recursion depth \(O(h)\).

Check if two trees are identical

same(a, b):
  if a == NULL and b == NULL: return true
  if a == NULL or  b == NULL: return false
  if a.key != b.key: return false
  return same(a.left, b.left) and same(a.right, b.right)

Expression tree evaluation (classic)

Postorder naturally evaluates bottom-up.

eval(u):
  if u is a number: return u.value
  x = eval(u.left)
  y = eval(u.right)
  return apply(u.op, x, y)
Explain complexity properly: for any “visit each node once” recursion, time is \(\Theta(n)\).

6) General rooted trees: left-child / right-sibling (lc/rs)

COMP2119 often represents a general tree node like this:

struct tree_node {
  int value;
  tree_node* lc; // leftmost child
  tree_node* rs; // right sibling
};

How to iterate children of a node

p = u.lc
while p != NULL:
  // p is a child of u
  p = p.rs

Example: sum values in a general tree

sum(u):
  if u == NULL: return 0
  total = u.value
  p = u.lc
  while p != NULL:
    total += sum(p)
    p = p.rs
  return total
Trap: lc points down; rs points sideways. Students often recurse on rs incorrectly (double-count) or forget it (miss siblings).

7) Proof templates (short, strict, and exam-correct)

A) Degree/edge counting template

Use these two equations (rooted tree):

\[ \sum_v ext{children}(v) = n-1,\quad \sum_i x_i = n \] where \(x_i\) is number of nodes with \(i\) children.

B) Full binary tree leaves proof (write like this)

Let \(I\) be internal nodes and \(L\) leaves. A full binary tree has exactly 2 outgoing child-edges from each internal node, so total edges \(E=2I\). But any tree has \(E=n-1=(I+L)-1\). Hence \(2I=I+L-1\Rightarrow L=I+1\).

C) “Every node visited once” runtime proof

  • Show each call does \(O(1)\) local work.
  • Each node triggers exactly one call.
  • Total time \(=O(1)\cdot n=\Theta(n)\).

8) Hard traps (this is where marks disappear)

Traversal mistakes

  • Mixing preorder vs inorder vs postorder (memorize the three sequences).
  • Not following the specified child visiting order.
  • For iterative inorder: forgetting the outer condition “u != NULL OR stack not empty”.

Reconstruction mistakes

  • Splitting the wrong side in inorder (root must appear exactly once).
  • Not using a global preorder index (or not passing it by reference).
  • Claiming pre+post is unique without extra conditions.
  • Forgetting to state complexity and how you achieved it.
Strict exam habit: when asked to “draw the tree”, also write the traversal(s) of your drawn tree to verify.

9) Hard practice set (COMP2119-style) — with answer key

A) Traversals

  1. Given the tree: root 5; left child 3 with children 1 and 4; right child 8 with children 7 and 9. Write preorder, inorder, postorder, level-order.
  2. Write an iterative inorder algorithm using one stack. State time and extra space.

B) Reconstruction (pre + in)

Preorder: 1,2,4,5,7,3,6,8,9,10. Inorder: 4,2,7,5,1,3,9,8,10,6.

  1. Draw the tree.
  2. Show the first two splits explicitly (write inorder left/right ranges).
  3. Give time complexity with and without a hashmap of inorder positions.

C) Proof

  1. Prove: in any non-empty full binary tree, \(L=I+1\).
  2. True/False (justify): Preorder + Postorder uniquely determines a binary tree.

D) Code-completion (very exam-like)

Node* buildTree(int inL, int inR, int& preIdx) {
  if (__________) return NULL;
  int rootVal = preorder[preIdx];
  preIdx++;
  int k = pos[rootVal];
  Node* root = new Node(rootVal);
  root->left  = buildTree(__________, __________, preIdx);
  root->right = buildTree(__________, __________, preIdx);
  return root;
}

Answer key (open only after attempting)

Show answers

A)

  • Preorder: 5,3,1,4,8,7,9
  • Inorder: 1,3,4,5,7,8,9
  • Postorder: 1,4,3,7,9,8,5
  • Level-order: 5,3,8,1,4,7,9
  • Iterative inorder: time \(O(n)\), extra space \(O(h)\).

B)

  • Root = 1 (from preorder). Inorder splits as left [4,2,7,5] and right [3,9,8,10,6].
  • Next root = 2. Inorder left part [4,2,7,5] splits as left [4] and right [7,5].
  • With hashmap \(pos\): total \(O(n)\). Without hashmap (linear scan for k): worst \(O(n^2)\).

C)

  • Proof via edges: \(E=2I\) and \(E=I+L-1\Rightarrow L=I+1\).
  • False in general; becomes true if the tree is known to be full (and labels distinct).

D)

  • Base case: inL > inR
  • Left: buildTree(inL, k-1, preIdx)
  • Right: buildTree(k+1, inR, preIdx)
